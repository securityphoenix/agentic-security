# NIST AI 600-1 scanner coverage map

**Framework version:** NIST AI 600-1 (2024) — Artificial Intelligence Risk Management Framework: Generative AI

This table maps NIST AI 600-1 controls to the scanner rules that provide evidence for each
control, or marks the control as "Out of scope" (requires human process) or "Not covered"
(no current scanner rule).

An `auditor-ready` compliance report is generated by `/compliance-report nist`. The scanner
can only provide automated evidence for controls that involve detectable code or configuration
patterns. Governance, organisational, and process controls require human attestation.

## Not covered — gaps requiring new rules or process

The following control families have zero scanner coverage and require human process:

- **GV-1.x**: Policy documentation, roles, executive oversight — pure governance
- **GV-2.x**: Workforce training — HR/L&D domain
- **GV-5.x**: Community engagement, bias/fairness — social/organisational
- **GV-6.5**: AI incident response plan — process documentation
- **MS-2.7**: AI output factuality (hallucination measurement) — requires golden-set evaluation
- **MS-3.3**: Protected-class bias testing — requires labelled demographic test sets
- **MG-1.3**: Residual risk acceptance — executive decision

## How to generate an attestation report

```bash
/compliance-report nist
```

The report maps scanner findings to the controls above and produces a
`nist-ai-600-1-attestation.md` file ready for auditor review. Controls with no
scanner evidence are listed as "Human attestation required."
